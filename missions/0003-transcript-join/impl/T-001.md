## What I built

### libs/core/src/transcript.ts (~180 lines)
Typed stream-parser for Claude Code jsonl. Full-load then line-split (files are ~1–10 MB in practice). Two layers:

**`parseTranscriptEvent(raw)`** — pure normalizer:
- Returns null if `.type` is missing (malformed event)
- Whitelist: `assistant | user | system | attachment | permission-mode` — unknown types still emit stub events for ordering but surface in `unknownTypes` counter
- On `assistant`: extracts `model`, `toolUseNames` (from `content[].tool_use.name`), and `usage` (input/output/cache_read/cache_creation, guarded with `|| 0`)
- Preserves `isSidechain` (distinguishes sub-agent events)

**`readTranscriptFile(path)`** — returns `{ events, skipped, unknownTypes }`:
- `skipped` = malformed JSON lines + events we couldn't normalize
- `unknownTypes` = `{ [typeString]: count }` for schema-drift warnings

**Discovery helpers** (all on top of the path-transform `workspaceHash`):
- `jsonlDir(workspacePath, home?)` → `~/.claude/projects/<hash>/`
- `findParentTranscript(ws, sid)` → absolute path or null
- `findSubAgentTranscripts(ws, parentSid)` → list of `{path, metaPath, agentId}` (empty if no subagents folder)
- `readSubAgentMeta(metaPath)` → `{agentType, description}` or null
- `workspaceHash(ws)` → literal `/` → `-` transform (exported for principle-14 integration)

### libs/core/src/join.ts (~130 lines)

**`joinMission(state, { workspacePath, homeOverride })`** — the typed port of SKILL.md §12 principle 13:

1. For each sid in `state.transcript_session_ids[]`:
   - Try to find parent jsonl → if missing, add to `missingTranscripts`, skip
   - Load + filter events to `[state.created, state.finished_at]` window
   - For each in-window `assistant` event: bump `turnCount`, collect model, sum usage, count tools
   - Discover + walk every sub-agent jsonl; if it has 0 in-window events, skip (sub-agent ran outside mission's window); otherwise collect its `agentType` + `description` from meta.json + turn count
2. Return `SessionSlice` with `window`, `sessionIds`, `missingTranscripts`, `parentFilesRead`, `subAgentFilesRead`, `turnCount`, `tokens{input,output,cache_read,cache_creation,total}`, `models[]`, `toolCounts{}`, `subAgents[]`.

Fail-open at every failure mode — never throws. Missing files surface as arrays; malformed JSON lines are skipped via `readTranscriptFile`'s counter.

## Decisions I made

- **Full-load vs streaming.** Went with `readFile + split` for jsonl. Files in practice are 1–10 MB; split is sub-100ms. Streaming adds complexity (readline, partial-line state) for no measured benefit at current scale.
- **No Zod yet.** Hand-coded coercion keeps the core lib under 200 lines per file. Zod pays off when we expose public input boundaries (CLI flags, IPC contracts in wave 5). Add then.
- **Failure mode is "report, not throw".** `missingTranscripts` array lets the UI render "X sessions had transcripts, Y didn't" without error-boundary acrobatics. This also aligns with principle 13's fail-open spec.
- **`<synthetic>` model strings aren't filtered.** Some assistant events in Claude Code's transcript have `.message.model === "<synthetic>"` for internal turns. Showed up in 0029 telemetry output alongside `claude-opus-4-7`. Left in — honest information about what's in the transcript, not noise. User can filter in presentation layer if they want.
- **Sub-agent window filter.** Sub-agents whose events are entirely outside the mission window are skipped from the `subAgents` list (not just given turnCount=0). Reduces noise in the drilldown.
- **Tool counts merged across parent + sub-agent.** Could have kept them separate. Merging gives the "what tools did this mission use" answer naturally; which agent used which is visible via the sub-agents block.
- **Top-8 tool-use truncation.** Enough to see the signal (Read/Edit/Bash are always top); avoids 30-tool walls in the CLI. Full list available via API consumers.
