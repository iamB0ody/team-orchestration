## What I built

### 1. `apps/dashboard-electron/icons/icon.svg` — master app icon

1024×1024 SVG, ~2 KB, source-of-truth for all platform outputs.

**Design spec:**
- Background: `#000000` (matches `--bg-0`, visual continuity with window chrome)
- Corner radius: 128px (12.5%) — macOS squircles on top anyway
- Accent frame: 2%-inset rounded rectangle in `#00a028` (`--accent-dim`) at 50% opacity for small-size shape definition
- Primary glyph: hand-tuned λ (lambda) path in `#00ff41` (`--accent`) — monospace-inspired contour, reads at 16×16
- Corner markers: 4 L-shaped clips in accent at 55% opacity — the "viewport" motif from the hacker-theme tokens

Brand rationale: λ has been the tot mark since mission 0034 (favicon seed) through the Angular app header (`shell__brand-accent`). Committed to it.

### 2. `apps/dashboard-electron/icons/README.md`

Documents the raster-output pipeline. At wave 7, `electron-builder` reads the master SVG and emits:
- `icons/mac/icon.icns` — macOS multi-size `.icns` bundle (1024/512/256/128/64/32/16)
- `icons/win/icon.ico` — Windows multi-size `.ico` (256/128/64/48/32/16)
- `icons/linux/*.png` — separate PNG per size for Linux desktop environments

Also captures the "don't change" list (bg color, glyph color, glyph shape) so the brand doesn't accidentally drift.

### 3. `apps/dashboard-electron/BRANDING-CHECKLIST.md` — 100+ line verifiable spec

This is the real deliverable. Covers every surface where default "Electron" branding leaks in a production build:

**Product identity table** — product name, full name, app ID, bundle id, AppUserModelId, repository, license, copyright.

**Wave 5 runtime checklist (19 items)** organized by surface:
- **App identity**: `app.setName("tot")`, `app.setAppUserModelId(…)`, `app.setAboutPanelOptions(…)` (replaces "About Electron")
- **User-data paths**: `userData`, `sessionData`, `logs`, crash-reporter dir — all overridden to `tot/` (default is `Electron/` which leaks into `~/Library/Application Support/`)
- **Window**: `title`, `backgroundColor: "#000000"` (no white flash), `icon`, `titleBarStyle: "hiddenInset"` on macOS, explicit dimensions, remembered via `electron-window-state`
- **Menu bar**: full `Menu.setApplicationMenu` replacement — app / edit / view / window / help menus spec'd; zero "Electron" / "Hide Electron" / "Quit Electron" strings anywhere
- **Dock/taskbar**: `app.dock.setIcon`, dock menu stub, Windows `AppUserModelId`, Linux `.desktop` fields
- **Protocol handler**: `app.setAsDefaultProtocolClient("tot")` for `tot://mission/0029`-style deep links (future)
- **Renderer title**: per-route titles via `Title` service, extending `app.routes.ts`
- **DevTools**: enabled in dev, stripped in production via NODE_ENV gate
- **IPC**: `contextBridge.exposeInMainWorld("api", …)`; no `ipcRenderer` leaks

**Wave 7 packaging checklist** — full `electron-builder.json` spec with:
- `appId: "com.iamB0ody.tot"`, `productName: "tot"`
- `artifactName: "tot-${version}-${os}-${arch}.${ext}"`
- mac: dmg + zip, icns path, hardenedRuntime, entitlements
- dmg: custom title, background, window size, layout coords
- win: NSIS + portable, publisherName, installer icons
- linux: AppImage + deb, desktop file with Categories, GenericName, Comment
- publish: github provider pointed at `iamB0ody/team-orchestration`

**Verification gates** — concrete checks to run at the wave-5 and wave-7 commit gates:

## Decisions I made

- **Ship SVG master without raster outputs.** User said "using svg for now". electron-builder ≥24 can auto-rasterize a master SVG into platform-specific bundles at package time — we don't need to pre-commit `.icns` / `.ico` / `.png`. Keeps the repo small + the source truly single-source-of-truth.
- **Empty `mac/`, `win/`, `linux/` subdirs are NOT committed.** electron-builder creates them on demand. The README explains where they go.
- **The λ glyph is hand-pathed, not `<text>`.** Text-based SVG icons render differently depending on platform font availability. Hand-paths are portable. Trade-off: harder to iterate; but the brand glyph is locked so this isn't a recurring cost.
- **Accent frame at 50% opacity.** At 1024px it's almost invisible, but at 16×16 it becomes the dominant shape cue (without the frame, a black-on-dark-dock icon blurs into the dock). Design verified mentally at multiple sizes.
- **Checklist is long on purpose.** User's explicit phrasing: "make fully sure of that". Ambiguity at wave 5 = missed item in production = "Electron" string somewhere. 40+ individual verification items means no room for slip.
- **`electron-builder.json` spec written in the checklist, not committed as a real file.** That's wave-7 work — the JSON only makes sense in context of real paths that exist (`release/`, icon outputs, entitlements). Pre-committing would be YAGNI.
- **Favicon is `<text>`-based, not hand-pathed.** Different trade-off from the master: favicons render at one tiny size in a browser, JetBrains Mono is loaded by the app already, and `<text>` is 1 line vs. a 10-line path. Acceptable.
